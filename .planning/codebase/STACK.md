# Technology Stack

**Analysis Date:** 2026-04-25

## Languages

**Primary:**
- Java 21 - Main application, rendering, ephemeris, configuration, scripting API, and command-line apps under `src/main/java`.

**Secondary:**
- Bash - Build/package helpers and resource download helpers in `mkPackage.bash`, `src/main/bash/createVersionFile.bash`, `src/main/bash/mkTestKernels.bash`, and `src/main/resources/resources/spice/spk/getSPK.bash`.
- Python 3 - Documentation packaging support in `mkPackage.bash` and model conversion tooling in `src/main/python/apps/convert_to_normalized_glb.py`.
- Groovy 3.0.25 - Embedded user scripting through JSR-223 in `src/main/java/kepplr/scripting/ScriptRunner.java`.
- GLSL / JME shader definitions - Rendering shaders and material definitions in `src/main/resources/kepplr/shaders` and `src/main/resources/assets`.
- reStructuredText - Sphinx documentation under `doc`.

## Runtime

**Environment:**
- Java 21+ runtime. `pom.xml` sets `<maven.compiler.release>21</maven.compiler.release>`, and `mkPackage.bash` enforces `REQUIRED_JAVA_VERSION=21`.
- Desktop OpenGL/LWJGL runtime through JMonkeyEngine. Runtime launch scripts generated by `mkPackage.bash` select native LWJGL jars by OS/architecture.
- JavaFX runtime. `pom.xml` has platform profiles for `mac-aarch64`, `mac`, `linux`, `linux-aarch64`, and `win` JavaFX classifiers.
- Python runtime for docs/package generation and offline model conversion. `mkPackage.bash` creates `venv` and installs Sphinx packages; `src/main/python/apps/convert_to_normalized_glb.py` expects Blender Python (`bpy`) for model conversion.

**Package Manager:**
- Maven - Build and dependency manager via `pom.xml`.
- Lockfile: missing. There is no `pom.lock`/Maven lock equivalent, package-lock, pnpm lock, or Python requirements lock in the repository.
- Python package installation is ad hoc in `mkPackage.bash` (`wheel`, `sphinx`, `sphinx-theme-pd`, `Pillow`), not pinned in a requirements file.

- Groovy JSR-223 3.0.25 - Scripting engine in `src/main/java/kepplr/scripting/ScriptRunner.java`.
- Sphinx - Documentation site generation from `doc/conf.py` and `doc/*.rst`.

**Build/Dev:**

## Configuration

**Environment:**
- Runtime configuration is file/property based through Jackfruit and Apache Commons Configuration, not environment-variable based. Core defaults live in `src/main/java/kepplr/config/KEPPLRConfigBlock.java`.
- Required runtime resource settings include `resourcesFolder` defaulting to `resources`, `outputRoot` defaulting to `output`, and `spice.metakernel` defaulting to `resources/spice/kepplr.tm`.
- SPICE metakernel configuration is modeled in `src/main/java/kepplr/config/SPICEBlock.java`; the packaged default metakernel is `src/main/resources/resources/spice/kepplr.tm`.
- Body and spacecraft resource paths are relative to `resourcesFolder`, configured by `src/main/java/kepplr/config/BodyBlock.java` and `src/main/java/kepplr/config/SpacecraftBlock.java`.
- Logging is configured through config properties `logLevel` and `logFormat` in `src/main/java/kepplr/config/KEPPLRConfigBlock.java`, applied by `src/main/java/kepplr/config/KEPPLRConfiguration.java`.
- No `.env` files or secret config files were detected at repo root or common shallow paths.

**Build:**
- `pom.xml` is the primary build configuration.
- `mkPackage.bash` is the distribution build script and generated launcher script template.
- `doc/conf.py`, `doc/make_doc.bash`, and `doc/make.bat` configure Sphinx documentation builds.
- `src/main/bash/createVersionFile.bash` is invoked by Maven to generate version metadata.
- `src/main/resources/resources/spice/kepplr.tm` is the default SPICE metakernel bundled as an application resource.
- `.codex/config.toml`, `.codex/agents/*.toml`, and `.codex/skills/*/SKILL.md` configure GSD/Codex project automation.

## Platform Requirements

**Development:**
- Java 21+.
- Maven 3.6.3+ enforced by `pom.xml`; `README.md` asks for Maven 3.8+ and `doc/index.rst` recommends Maven 3.9.6+.
- SPICE kernels must be present for full runtime ephemeris behavior. The default metakernel references kernels under `resources/spice`; large SPK kernels can be downloaded with `src/main/resources/resources/spice/spk/getSPK.bash`.
- Desktop graphics environment for JME/LWJGL rendering. Render tests configure `GLFW_PLATFORM=x11` and `LIBGL_ALWAYS_SOFTWARE=1` in `pom.xml`.
- Python 3 for docs packaging and optional model tools; Blender 5.0+ is documented for `src/main/python/apps/convert_to_normalized_glb.py`.

**Production:**
- Desktop Java distribution generated by `./mkPackage.bash`; packages include launch scripts, `lib/`, docs, and Python tools.
- Supported platforms are macOS Intel, macOS Apple Silicon, Linux x86_64, Linux aarch64, and Windows via JavaFX platform profiles in `pom.xml`.
- No server, cloud deployment target, container, or hosted production environment is detected.

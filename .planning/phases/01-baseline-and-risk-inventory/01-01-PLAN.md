# Plan 01: Capture Baseline Command Status

<objective>
Establish the pre-change health baseline for Phase 1 by running the agreed command set and recording exact outcomes in a dedicated planning artifact.
</objective>

<threat_model>
This plan runs local Maven checks and writes a markdown report. It does not execute untrusted scripts, modify source files, or read secrets. Main risks are long-running commands, noisy build output, and accidentally treating pre-existing failures as newly introduced work.

Mitigations:
- Run only the agreed baseline commands: `mvn test` and `mvn spotless:check`.
- Record command, exit status, and concise failure summary rather than copying excessive logs.
- Do not modify production code to fix failures in this plan.
- Do not include environment secrets or local absolute credential paths in the report.
</threat_model>

<must_haves>
<truths>
- D-01: Phase 1 baseline scope is `mvn test` and `mvn spotless:check`.
- D-02: Pre-existing failures are baseline debt and must be documented exactly.
- D-03: Live render Maven execution is not required in Phase 1; inspect and document the configured render flow instead.
- D-04: Phase 1 implementation must produce an explicit checklist of baseline commands, results, failures, and gaps.
</truths>
<outputs>
- `.planning/phases/01-baseline-and-risk-inventory/01-BASELINE.md` exists.
- The baseline report contains sections for `mvn test`, `mvn spotless:check`, and render-flow inspection.
</outputs>
</must_haves>
